## Bug 修复（5个）

### Bug 1: 上传文件报 `Table 'student_management.upload_sessions' doesn't exist`
- **原因**: 后端 files.js 启动时 `CREATE TABLE IF NOT EXISTS` 的 `.catch(() => {})` 静默吞掉错误，表实际未创建
- **修复**: 在服务器 MySQL 上手动执行 CREATE TABLE 创建 `upload_sessions` 和 `upload_chunks` 表

### Bug 2: 下载较大文件返回 HTTP 413
- **原因**: 后端 `/files/:id/download` 接口对超过 2MB 的文件直接返回 413 状态码和 JSON 提示，前端 `downloadFileStreaming` 用 fetch 拿到非 200 就抛错
- **修复**: 移除 2MB 大小限制和 413 返回逻辑，改为直接流式返回二进制数据（Content-Type + Content-Disposition + Content-Length），所有文件统一走二进制流下载

### Bug 3: 论坛管理页进入即显示"已取消置顶"
- **原因**: `el-switch` 的 `v-model` 绑定 MySQL 返回的整数 `pinned`（0/1），switch 内部类型转换时触发 `@change` 事件，导致页面加载就调用 `handlePin`
- **修复**: 改用 `:model-value="!!row.pinned"` 单向绑定 + `@change="(val) => handlePin(row, val)"` 显式传值，避免初始渲染触发 change

### Bug 4: 学生端通知点击后已读状态不更新、红点不消失
- **原因**: `StudentDashboard.vue` 的 `showNotifDetail` 只弹出详情对话框，没有调用 `markAsRead` API
- **修复**: 在 `showNotifDetail` 中对 `type === 'notification'` 且 `!is_read` 的项调用 `markAsRead(item.raw.id)`，更新本地 `notifications` 数组中对应项的 `is_read = true`，并递减 `unreadCount`

### Bug 5: 管理端考试添加本日考试会显示到前一日
- **原因**: MySQL `DATE` 类型经 mysql2 driver 返回为 JS Date 对象（UTC 午夜），`handleEdit` 直接将 Date 对象赋给 `el-date-picker` 的 v-model，时区转换导致日期偏移
- **修复**: 
  - 前端 `handleEdit` 中将 Date 对象转为 `YYYY-MM-DD` 字符串（使用本地时间 getFullYear/getMonth/getDate）
  - 后端 exams.js 所有 SELECT 查询用 `DATE_FORMAT(exam_date, '%Y-%m-%d')` 返回字符串，彻底避免 Date 对象时区问题

## 大文件上传超时修复

### 根因分析
上传大文件进度到 100% 后卡住并显示"请求超时"，根本原因是多层面的超时配置不够：

1. **Node.js HTTP server 默认超时太短**（120s），而 complete 合并步骤对大文件需要较长时间
2. **前端分片上传并发控制 Bug**：`Promise.race(chunkPromises)` 不正确——已完成的 Promise 留在数组中导致并发控制失效，所有分片实际上同时发送
3. **study_materials.js 通知发送低效**：对班级所有学生逐条 INSERT 通知（12 条 = 12 次查询），合并步骤很慢
4. **uploadLimiter 限制过严**：30 req/min 对大文件分片上传（1 init + N chunks + 1 complete）可能不够

### 修复内容
1. **index.js**: 设置 `httpServer.timeout = 300000`（5分钟），添加 `keepAliveTimeout` 和 `headersTimeout`
2. **study_materials.js**: 通知改为批量 INSERT（`VALUES ?` + 嵌套数组），12 次查询 → 1 次
3. **file.js / study_material.js**: 修复并发控制（用 executing 数组 + splice 移除已完成 Promise），进度条改为 95% 表示分片完成、100% 表示合并完成
4. **index.js**: uploadLimiter 从 30/min 增加到 60/min

### 服务器配置确认
- Nginx: `proxy_read_timeout 300s`, `client_max_body_size 50m` ✓（已配置）
- MySQL: `max_allowed_packet = 1GB` ✓（足够大）
- Node: `httpServer.timeout = 300s` ✓（新增）
- PM2: online, 健康检查 200 ✓
