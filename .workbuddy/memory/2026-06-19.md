## Nginx崩溃修复（resty.core模块找不到）

**问题**：宝塔Nginx启动失败，报错 `no file 'resty/core.lua'`，导致phpMyAdmin 888端口无法访问，前端80端口也挂了。

**根因**：宝塔编译的nginx带了lua_nginx_module，启动时自动加载resty.core，但lua库路径（`/www/server/nginx/lib/lua/`）没有配置到nginx.conf中。宝塔的init.d脚本会在 `include proxy.conf;` 这一行后面自动注入 `lua_package_path`，但nginx.conf里缺少这行。

**修复步骤**：
1. 在nginx.conf的http块加上 `include proxy.conf;`（proxy.conf存在于`/www/server/nginx/conf/`）
2. 用 `/etc/init.d/nginx start` 启动（而非systemctl），宝塔脚本自动注入lua_package_path
3. 修复 phpmyadmin.conf：目录名改为实际的 `phpmyadmin_0cc518789e4f3bc0`，sock改为 `php-cgi-81.sock`

**结果**：80端口200，888端口phpMyAdmin 200，API正常，PM2 online

## 关键文件
- `/www/server/nginx/conf/nginx.conf` - 加了 include proxy.conf
